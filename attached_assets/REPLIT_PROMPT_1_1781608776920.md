## 4. FIX THE LIVE TICKER COVERING PAGE CONTENT

The live scrolling ticker/marquee at the top of the app (showing fuel prices like "95 область 59₽ −0% ◆ ДТ область 67₽ −0%") overlaps the page content on ALL pages. This is a critical layout bug.

**Root cause**: The ticker is likely `position: fixed` or `position: sticky` at the top, but the page content containers don't have enough `padding-top` or `margin-top` to push content below the ticker.

**Fix**:
1. Measure the ticker height (approximately 36–44px)
2. In the main layout wrapper or each page's scroll container, add:
   ```css
   padding-top: calc(<ticker-height>px + env(safe-area-inset-top));
   ```

   where `<ticker-height>` is the actual rendered height of the ticker bar
3. OR: Place the ticker inside the normal document flow (not `position: fixed`), so it naturally pushes content down — but keep it visually at the top
4. The ticker must appear BELOW the Telegram mini app header ("Back" / "Close" buttons row), not overlapping it
5. On the News page specifically: the "КРИЗИСНЫЙ РЕЖИМ · Уровень 5/5" banner must appear below the ticker, not overlapping it
6. Test on all 5 tab screens (Карта, Талоны, ИИ-Советник, Игры, Новости): each must have its content starting below the ticker
7. Make sure the ticker itself does not overlap the Telegram WebApp safe area at the top
