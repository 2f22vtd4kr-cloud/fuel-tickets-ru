Full Replit .replit + start.sh fix for Autoscale deployment (free plan)
Copy-paste these exactly into your repo (edit via GitHub or Replit Files tab), commit/push, then go to Tools → Publishing → Autoscale and click Publish (or Redeploy). This eliminates the old Node.js api-server entirely.
1. Replace entire `.replit` content with this:
modules = ["nodejs-20", "python-3.12", "web"]

[deployment]
router = "application"
deploymentTarget = "autoscale"
build = ["bash", "-c", "pip install -r requirements.txt && pnpm install --frozen-lockfile && pnpm --filter @workspace/tma-frontend run build"]
run = ["bash", "start.sh"]

[deployment.postBuild]
args = ["pnpm", "store", "prune"]
env = { "CI" = "true" }

[workflows]
runButton = "Project"

[userenv]
[userenv.production]
TMA_PORT = "8080"

[nix]
channel = "stable-25_05"

# Optional: keep a few useful ports for local dev
[[ports]]
localPort = 8080
externalPort = 80

[[ports]]
localPort = 8000
externalPort = 8000
2. Replace entire `start.sh` with this improved version:
#!/bin/bash

# Do NOT use 'set -e' — bot failure must not kill FastAPI in production

echo "=== Starting TMA Application on Replit Autoscale ==="

# Run bot in background (polling mode to avoid unreachable webhook port)
echo "Starting Telegram Bot (polling)..."
unset REPLIT_DEPLOYMENT  # Force polling + clean stale webhook if any
python bot.py &
BOT_PID=$!

echo "Starting FastAPI Backend (serves React frontend) on port ${TMA_PORT:-8080}..."
exec python -m tma_backend.main
3. Post-Deploy Steps in Replit UI
	1	Clear old build cache:
	◦	In Replit Shell: rm -rf artifacts/tma-frontend/dist artifacts/api-server node_modules ~/.cache/pnpm
	◦	Or just delete the entire artifacts/ folder and let build recreate it.
	2	Go to Tools → Publishing.
	3	Select Autoscale (only option on free plan).
	4	Set up / Publish (or Redeploy if already configured).
	5	Monitor Deployment Logs in real-time.
Expected Successful Logs (look for these)
	•	pnpm install + pnpm --filter ... run build succeeds
	•	Starting Telegram Bot (polling)...
	•	Starting FastAPI Backend...
	•	Uvicorn/FastAPI binding to 0.0.0.0:8080
	•	No mention of artifacts/api-server or Node.js errors
Why This Works on Autoscale
	•	deploymentTarget = "autoscale" + router = "application" tells Replit to treat it as a dynamic web app (not static).
	•	Build only does Python + React frontend (no dead Node api-server).
	•	start.sh runs both processes correctly; FastAPI serves static dist/ with SPA fallback (html=True).
	•	Port 8080 is respected via TMA_PORT and exposed properly.
	•	Bot uses polling (safer on Replit).
After publishing, test:
	•	Root URL → React app loads
	•	/api/... endpoints work
	•	Bot responds in Telegram
If build still fails, share the new deployment logs here. This should resolve the 404/no artifacts issue permanently.
